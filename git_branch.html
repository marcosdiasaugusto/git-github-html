Branches

O master é o branch principal do GIT.

O HEAD é um ponteiro especial que indica qual é o branch atual. Por padrão, o HEAD aponta 
para o branch principal, o master.

Criando um novo branch

git branch bug-123

Trocando para um branch existente

git checkout bug-123

Neste caso, o ponteiro principal HEAD esta apontando para o branch chamado bug-123.

Criar um novo branch e trocar

git checkout -b bug-456

Voltar para o branch principal (master)

git checkout master

Resolver merge entre os branches

git merge bug-123

Para realizar o merge, é necessário estar no branch que deverá receber as alterações. 
O merge pode automático ou manual. O merge automático será feito em arquivos textos que 
não sofreram alterações nas mesmas linhas, já o merge manual será feito em arquivos textos 
que sofreram alterações nas mesmas linhas.

A mensagem indicando um merge manual será:

Automerging meu_arquivo.txt

CONFLICT (content): Merge conflict in meu_arquivo.txt

Automatic merge failed; fix conflicts and then commit the result.

Apagando um branch

git branch -d bug-123

Listar branches

git branch

Listar branches com informações dos últimos commits

git branch -v

Listar branches que já foram fundidos (merged) com o master

git branch --merged

Listar branches que não foram fundidos (merged) com o master

git branch --no-merged

Criando branches no repositório remoto

Criando um branch remoto com o mesmo nome

git push origin bug-123

Criando um branch remoto com nome diferente

git push origin bug-123:new-branch

Baixar um branch remoto para edição

git checkout -b bug-123 origin/bug-123

Apagar branch remoto

git push origin:bug-123

Um branch nada mais é do que uma ramificação do projeto.
                                                                       b---- branch funcionalidade
            base
--------\b-----------------------------------------/b-------- projeto (ramo principal)
                                                                        base
             b----- branch funcionalidadeA
ponto b base comum, exato momento em que é criado a branch. Onde o ramo principal máster 
branch máster e o novo ramo fazem a ramificação.

ramificação/galho da arvore/projeto
cada traço é um commit (snapshop, fotografia) no seu projeto, linha do tempo
a linha principal é chamada de branch máster, ramo principal, linha comum do seu projeto
podemos criar novas branchs para testar ou adicionar uma nova funcionalidade extra ao nosso 
projeto, trabalhando em um projeto de forma colaborativa, com vários desenvolvedores, cada 
desenvolvedor pode trabalhar em cada um do seu branch e quando terminar o trbalho submeter 
ao branch principal.  

Criar o arquivo .gitignore e acresentar o info.txt para este arquivo não ser versionado.

Podemos criar um branch sempre quando acharmos necessário. Testar uma implementação 
diferente, testar um recurso mas não quero mexer na branch principal (máster), cria e ativa 
um novo branch, cria seus arquivos ali, talvez você não goste, deleta o branch criado e 
volta para o branch principal e continua trabalhando. Melhor que trabalhar somente no 
principal, evita muitos problemas. Dá o nome do branch da funcionalidade que você for 
fazer.

Observe sempre o * que informa qual a branch que estamos

A funcionalidadeA é a branch atual (* e cor verde) e contem todos os commits da máster até 
o momento da sua criação.
Pereceba que o arquivoA.html faz parte da outra branch, por isso não aparecer no branch máster.
Após realização dos testes pode-se excluir a branch.
