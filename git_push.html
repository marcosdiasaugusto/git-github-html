Enviar arquivos/diretórios para o repositório remoto

O primeiro push de um repositório deve conter o nome do repositório remoto e o branch.

git push -u origin master

Os demais pushes não precisam dessa informação

git push

Código de compartilhamento

Para compartilhar seu código, você cria um repositório em um servidor remoto para o qual copiará seu repositório local.

Para minimizar o uso de espaço no servidor remoto, você cria um repositório vazio: um que tem apenas o .git
objetos e não cria uma cópia de trabalho no sistema de arquivos. Como um bônus, você configurou este controle remoto 
como um servidor upstream para compartilhar atualizações facilmente com outros programadores.

No servidor remoto:

git init --bare /path/to/repo.git

Na máquina local:

git remote add origin ssh: // username @ server: /path/to/repo.git

(Observe que ssh: é apenas uma forma possível de acessar o repositório remoto.)

Agora copie seu repositório local para o remoto:

git push --set-upstream origin master

Adicionar --set-upstream (ou -u) criou uma referência upstream (rastreamento) que é usada pelo Git sem argumentos
comandos, por exemplo git pull.
